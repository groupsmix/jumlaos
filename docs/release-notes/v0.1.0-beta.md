# JumlaOS v0.1.0-beta — Mali Private Beta

> Release date: 2026-05-10
> Launch label: **JumlaOS Mali Private Beta**

## What ships

The **Mali** module is the only customer-facing module in this release:

- Debtors: create / edit / list / soft-delete, ICE field, blocked flag.
- Debt events: credit, payment, adjustment, with immutable audit history.
- Payments: cash, transfer, cheque, with idempotency keys.
- Invoices: gap-free numbering, finalization lock, immutable after finalization.
- Invoice PDF: bilingual ar-MA / fr-MA, Cairo + Amiri fonts, WeasyPrint.
- DGI tax export: per-period CSV / JSON, 10-year retention.
- Phone-OTP login (WhatsApp + SMS fallback).
- RBAC: owner / accountant / driver.
- Postgres RLS multi-tenant isolation, validated by integration tests.

## What is hidden / disabled

| Module    | Status        | How it's hidden                                               |
| --------- | ------------- | ------------------------------------------------------------- |
| Talab     | Not ready     | `modules_enabled.talab = false` per business; routes 403/501  |
| Makhzen   | Not ready     | `modules_enabled.makhzen = false` per business; routes 403/501|
| Analytics | Partial only  | Not exposed in production navigation                          |

The web UI hides nav entries for any module whose flag is `false` on `/v1/me`.

## Migration notes

Fresh install:

```
docker compose -f infra/docker-compose.dev.yml up -d
(cd apps/api && uv run alembic upgrade head)
(cd apps/api && uv run python -m jumlaos.scripts.seed_demo)
```

There is no upgrade path from a previous release because this is the first tagged version.

## Required production secrets

See `docs/deployment.md` and `.env.example`. At minimum:

- `JUMLAOS_SECRET_KEY` (≥32 chars, random)
- `DATABASE_URL` and `DATABASE_URL_SYNC` (Neon, SSL required)
- `REDIS_URL` (Upstash, non-localhost)
- `JUMLAOS_ALLOWED_ORIGINS` (production web origin)
- `OTP_TRANSPORT=whatsapp` + `WHATSAPP_PHONE_NUMBER_ID` + `WHATSAPP_ACCESS_TOKEN` + `WHATSAPP_APP_SECRET` + `WHATSAPP_WEBHOOK_VERIFY_TOKEN`
- `R2_ENDPOINT` + `R2_BUCKET` + `R2_ACCESS_KEY_ID` + `R2_SECRET_ACCESS_KEY`
- `SENTRY_DSN`

The API will refuse to boot in `prod` / `staging` if any of the above are missing or set to dev defaults.

## Known issues

- Talab and Makhzen API routes exist as stubs and return `status: "not_yet_enabled"`. Do not market these modules as available.
- Frontend offline mode is best-effort only; do not rely on it for critical flows.
- DGI export format is provisional and pending final validation against the 2026 DGI schema.
